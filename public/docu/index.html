Classifier¶

class Classifier.Classifier¶

Class that is taking a trained Machine Learning algorithm to train different Classifier algorithms. The training is done in the train function while the validation is done in the function test

test()¶

Function to validate the trained classifier against the first 3000 cases of all five scenarios. Creates as output a error matrix that displays the files that were predicted wrongly. Gives also the possibility to test against individual files afterwards

Args:

folder (string): The directory of the trained ML Algorithm

train()¶

This function downloads the first 1000 files from the server for 5 different scenarios. The scenarios are Breakage in Line 1,9,12 and 19 and without any breakage. It then predicts the whole file and finde the error ratings for four different error scores. It stores the findings in a csv file

Args:

config (string): The directory of the trained ML Algorithm

class Objects.ResultFile(data, config=None, verbose=0)¶

Object used to calculate Error scores and display graphically the prevision and simulation.

calculateResults()¶

The file is comparing the simulated ship movement against the predicted ship movement. It seperates the Timeline in error windows that have the size of two previsions. For each window different Error scores are calculated. The Error scores are RMSE, Absolute Error Deviation and Mean Error. Afterwards the errorwindows are compared to each other and the highest error is selected as beeing representative for the file.

Returns:

numy Array (Error type, features): returns the maximum error ratings selected for this file as an Array

class Objects.Scaler¶

Scaler that Scales every column individually to an area between 0 and 1

fit(dat)¶

This function is used to fit the scaler to the data and must be called before scaling the data. It should only be called once to garanty data consistency and scaling with the same values. This Scaler scales each column individually to a value range between zero and one to have higher acuracy :param data: contains the data set that is used to scale the algorithm

Loads the scaler data of a already fitted scaler to use the Scaler for scaling testdata. Avoid fitting the Scaler again! For consistency the same scaler metadata should be used for testing and training :return: the scaler with the preloaded Metadata. This scaler does not need to be fitted!

Testing¶

Testing.Testing(metapath)¶

This function is loading trained metadata and is doing a prevision for a selected Fileindex. The prevision is done every n Timesteps so that each Timestep is only prevised once. The errors are then calculated based on the difference between prevision and simulation. The difference between these two is also shown graphically.

Args:

metapath (str): path to the metadata that should be tested

Testing.computeResult(file, loader, algo, scaler, config)¶

Function to calculate the Error values for a given file and model. The data is first scaled and then split into Testing sets. These Testing sets are then predicted by the algorithm and the previsions are grouped into error windows of two previsions. For each error window an Error Score is calculated and the window that has the highest error score is selelted to represent the error score of the whole file.

Args:

file (str): abbreviated filepath loader (SimulationFileLoader): Loader object that is used to load the file algo (Algorithm): The algorithm to be used to predict the File scaler (Scaler): The Scaler to be used to scale the File config (Configuration): Configurationobject containing the Configuration of the Algorithm

Returns:

numpy Array: Array containing the Error Score of the file
